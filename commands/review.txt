## Exit Criteria (must ALL be true before proceeding to /ship)
- [ ] Code reviewed across all 5 axes (correctness, readability, architecture, security, performance)
- [ ] All Critical and Important findings addressed
- [ ] No security vulnerabilities remaining
- [ ] User has reviewed findings

## Anti-Rationalization Table
| Rationalization | Reality |
|---|---|
| "We don't need a formal review" | Informal reviews miss things. Formal reviews catch patterns. |
| "It's a small change, review is overkill" | Small changes introduce bugs too. Especially small changes. |
| "The tests pass, so the code is fine" | Tests verify behavior. Reviews verify design. Both matter. |

## Red Flags
- Review only checked one axis (e.g., correctness but not security)
- Critical findings deferred without justification
- No specific file:line references in findings
