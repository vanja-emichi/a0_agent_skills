Load the skill `skills_tool:load incremental-implementation` and follow its instructions.

Also load `skills_tool:load test-driven-development` for the TDD cycle within each task.

Pick the next pending task from the plan. For each task:

1. Read the task's acceptance criteria
2. Load relevant context (existing code, patterns, types)
3. Write a failing test for the expected behavior (RED)
4. Implement the minimum code to pass the test (GREEN)
5. Run the full test suite to check for regressions
6. Run the build to verify compilation
7. Commit with a descriptive message
8. Mark the task complete and move to the next one

If any step fails, load `skills_tool:load debugging-and-error-recovery` and follow its recovery process.

---

⚠️ **MANDATORY** — If an active plan exists (check with `lifecycle:status`), wrap each task in phase lifecycle calls:

```
lifecycle:status  # Before starting the task
# ... implement task ...
lifecycle:status  # After completing the task
```

If `lifecycle:status` returns no active plan, skip phase calls gracefully — do not block on plan tool absence.
If a phase call fails, resolve the error before proceeding with the task.

## Exit Criteria (must ALL be true before proceeding to /test)
- [ ] All planned tasks implemented
- [ ] All unit tests pass
- [ ] No TODO comments remaining in new code
- [ ] Code compiles/builds without errors

## Anti-Rationalization Table
| Rationalization | Reality |
|---|---|
| "I'll test it all at the end" | Bugs compound. Test each increment. |
| "It works on my machine" | Prove it with automated tests, not manual checks. |
| "This refactor is too small to test separately" | Small untested changes become large untested codebases. |

## Red Flags
- More than 100 lines written without running tests
- Multiple unrelated changes in one increment
- Build broken between increments
