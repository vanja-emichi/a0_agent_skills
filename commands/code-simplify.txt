| Rationalization | Reality |
|---|---|
| "Simplification is cosmetic" | Simplification reduces bugs and improves maintainability. |
| "I can simplify and refactor in one pass" | Incremental simplification with tests is safer. |
| "The tests will still pass without running them" | Run tests after EVERY simplification. No exceptions. |
