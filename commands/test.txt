The specialist agent will load the test-driven-development skill and execute the full TDD cycle.
For new features: RED (failing test) → GREEN (minimal implementation) → REFACTOR.
For bug fixes: Prove-It pattern (reproduce with failing test → fix → verify).


## Exit Criteria (must ALL be true before proceeding to /review)
- [ ] All tests pass (unit, integration)
- [ ] Test coverage hasn't decreased
- [ ] Edge cases covered
- [ ] Bug fixes include reproduction tests

## Anti-Rationalization Table
| Rationalization | Reality |
|---|---|
| "It works, tests are just ceremony" | Tests are proof. "Seems right" is not done. |
| "Edge cases don't happen in practice" | Edge cases are where the worst bugs live. |
| "Manual testing is enough" | Manual testing doesn't persist. Tomorrow's change might break it. |

## Red Flags
- Tests that pass on first run (may not test what you think)
- Bug fixes without reproduction tests
- Skipped or disabled tests
